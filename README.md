本仓库包含《基于双臂任务感知适配的视觉-语言-动作机器人饮品服务系统》中文技术报告初稿。

- `report.html`：带排版样式的报告源文档，可直接用浏览器打开。

报告围绕 VLA、TinyVLA、ACT/ALOHA、双臂解耦、motion mask、阶段感知模仿学习等论文依据，提出面向智元 G1 双臂服务机器人的 Bimanual Task-aware Adapter（BTA）设计。

当前设计主流思路：# Codex 写作提示词：BTA-Lite + BTA-Loss 方法设计

## 1. 任务角色

你是一位擅长机器人学习、具身智能、VLA 模型和中文技术报告写作的学术编辑。请帮助我围绕研究生技术竞赛论文/技术报告，设计并润色一个方法章节，主题为：

**BTA-Lite + BTA-Loss：面向双臂饮品服务任务的任务感知适配方法**

请注意：当前阶段以**方法设计与论证**为主，不要虚构已经完成的实验结果，不要夸大为“全新大模型”，而是强调在 GR00T 类 VLA 基础模型上的轻量级任务适配与训练策略优化。

---

## 2. 项目背景

本项目面向具身智能与智能机器人系统创新设计，机器人平台为双臂机器人，任务场景为饮品服务。

核心任务流程包括：

1. 右臂抓取饮料瓶；
2. 右臂将饮料倒入杯子约三分之一；
3. 左臂抓取杯子；
4. 左臂将杯子放置到杯垫上；
5. 在部分阶段中，左右臂可能存在非对称协同关系，例如左臂稳定杯子，右臂执行倒水。

底层模型采用 GR00T 类视觉-语言-动作模型，输入包括图像、语言指令和机器人状态，输出为未来一段时间的 action chunk。

---

## 3. 当前问题

现有 VLA/GR00T 类模型在双臂任务微调时，通常将左右臂和夹爪动作拼接成一个整体动作向量进行模仿学习，例如：

[
A_t = [a_t^L, g_t^L, a_t^R, g_t^R]
]

其中 (a_t^L) 和 (a_t^R) 分别表示左臂和右臂动作，(g_t^L) 和 (g_t^R) 分别表示左右夹爪动作。

这种整体式 action chunk 监督方式存在以下问题：

1. **缺少左右臂角色建模**
   模型只知道要完成任务，但不显式知道当前是右臂主导、左臂主导，还是双臂协同。

2. **左右臂学习不均衡**
   整体 loss 下降并不代表左臂和右臂都学好了，可能出现某一侧手臂效果较好、另一侧较差的情况。

3. **单臂主导任务容易被无关动作干扰**
   例如右臂抓饮料时，左臂动作可能并不是关键监督目标；如果所有动作维度平均学习，会增加无关维度干扰。

4. **长时序任务阶段切换困难**
   倒水任务包含抓取、对准、倾倒、恢复、放置等阶段。如果缺少阶段感知，模型容易出现抓取成功但倒水不稳、倒水完成后姿态衔接不自然等问题。

5. **失败原因难以诊断**
   单一总 loss 难以判断失败来自右臂抓取、右臂倒水、左臂抓杯、左臂放置，还是双臂协同阶段。

---

## 4. 方法总思想

请将方法设计为：

**从“完整任务端到端模仿学习”升级为“基于任务流程先验的弱监督阶段标注 + 子技能均衡训练 + 双臂任务感知加权损失”。**

核心方法命名为：

**BTA：Bimanual Task-aware Adapter，双臂任务感知适配模块**

当前主要实现包括两个部分：

1. **BTA-Lite**
   不大幅修改模型结构，主要通过任务流程切分、task role token、阶段提示和子任务均衡采样，让模型显式区分不同子技能和左右臂角色。

2. **BTA-Loss**
   在训练损失中引入左右臂动作子空间划分和角色感知加权，使当前子技能重点监督关键手臂和关键夹爪动作。

---

## 5. BTA-Lite 设计

### 5.1 基于任务流程先验的弱监督阶段标注

请将数据集标注方式表述为：

**基于任务流程先验的弱监督阶段标注，而不是完全自监督。**

由于原始任务本身具有明确流程，可以先根据时间段或关键帧进行粗分段，再结合夹爪开合、机械臂末端运动变化、瓶身倾斜角度等信号进行半自动校正。

建议子任务划分为：

| 子任务     | Task Role Token      | 说明             |
| ------- | -------------------- | -------------- |
| 右臂抓饮料   | `RIGHT_GRASP_BOTTLE` | 右臂靠近并抓取饮料瓶     |
| 右臂对准并倒水 | `RIGHT_ALIGN_POUR`   | 右臂将瓶口对准杯口并完成倾倒 |
| 左臂抓杯子   | `LEFT_GRASP_CUP`     | 左臂抓取杯子         |
| 左臂放杯子   | `LEFT_PLACE_CUP`     | 左臂将杯子放置到杯垫     |
| 双臂协同倒水  | `BIMANUAL_HOLD_POUR` | 左臂稳定杯子，右臂执行倒水  |

可写为：

[
\tau = {\tau_1,\tau_2,\tau_3,\tau_4,\tau_5}
]

其中 (\tau_i) 表示由完整轨迹切分得到的第 (i) 个子技能片段。

### 5.2 Task Role Token

为不同子任务加入明确的任务角色标记，使同一个 VLA 模型能够根据角色条件输出不同动作模式。

示例：

```text
Task role: RIGHT_GRASP_BOTTLE. The right arm grasps the drink bottle.
```

```text
Task role: BIMANUAL_HOLD_POUR. The left arm stabilizes the cup, and the right arm pours the drink into the cup.
```

模型可形式化表示为：

[
\hat{A}=f_{\theta}(I, s, l, r)
]

其中：

* (I)：视觉输入；
* (s)：机器人状态；
* (l)：自然语言指令；
* (r)：task role token；
* (\hat{A})：预测动作序列。

### 5.3 子技能均衡采样训练

请强调：不是按顺序训练多个子技能，否则容易出现灾难性遗忘。

错误方式：

[
Task_1 \rightarrow Task_2 \rightarrow Task_3 \rightarrow Task_4
]

推荐方式：

[
Batch = {Task_1, Task_2, Task_3, Task_4, Task_5}
]

即把五类子任务放入同一训练集，通过 task role token 区分任务，并进行均衡采样训练。

可写为：

[
\mathcal{D}_{sub}
=================

\mathcal{D}*{RG}
\cup
\mathcal{D}*{RP}
\cup
\mathcal{D}*{LG}
\cup
\mathcal{D}*{LP}
\cup
\mathcal{D}_{BP}
]

训练时尽量保证：

[
P(Task_i)=\frac{1}{N}
]

从而避免某一类子任务样本过多导致模型偏向某一侧手臂或某一类动作。

## 6. BTA-Loss 设计

### 6.1 动作子空间划分

将整体动作向量划分为左臂、右臂、左夹爪和右夹爪四个子空间：

[
A_t = [a_t^L, g_t^L, a_t^R, g_t^R]
]

对应计算分项损失：

[
\mathcal{L}*L,\quad
\mathcal{L}*R,\quad
\mathcal{L}*{gL},\quad
\mathcal{L}*{gR}
]

其中：

* (\mathcal{L}_L)：左臂动作误差；
* (\mathcal{L}_R)：右臂动作误差；
* (\mathcal{L}_{gL})：左夹爪动作误差；
* (\mathcal{L}_{gR})：右夹爪动作误差。

### 6.2 角色感知加权损失

整体损失函数设计为：

[
\mathcal{L}_{BTA}
=================

w_L\mathcal{L}*L+
w_R\mathcal{L}*R+
w*{gL}\mathcal{L}*{gL}+
w_{gR}\mathcal{L}*{gR}
+
\lambda_s\mathcal{L}*{smooth}
]

其中：

* (w_L,w_R,w_{gL},w_{gR})：由当前任务角色和阶段决定的动态权重；
* (\mathcal{L}_{smooth})：可选动作平滑项，用于提升倒水过程中的动作稳定性；
* (\lambda_s)：平滑损失权重。

建议权重设计如下：

| 子任务                  | 左臂权重 | 右臂权重 | 左夹爪权重 | 右夹爪权重 |
| -------------------- | ---: | ---: | ----: | ----: |
| `RIGHT_GRASP_BOTTLE` |  0.3 |  1.0 |   0.2 |   1.2 |
| `RIGHT_ALIGN_POUR`   |  0.5 |  1.2 |   0.5 |   0.8 |
| `LEFT_GRASP_CUP`     |  1.0 |  0.3 |   1.2 |   0.2 |
| `LEFT_PLACE_CUP`     |  1.0 |  0.3 |   1.0 |   0.2 |
| `BIMANUAL_HOLD_POUR` |  0.8 |  1.2 |   0.8 |   0.8 |

请注意：

1. 非主导手臂权重不要设为 0，否则可能完全忽略辅助稳定动作；
2. 加权损失的目的不是让模型只学某一只手，而是让当前子技能重点学习关键动作维度；
3. 该损失同时具有训练约束和诊断价值，可以用于分析左臂、右臂、夹爪分别学习得如何。

## 7. 完整轨迹组合训练

在子技能均衡训练后，需要使用完整任务轨迹进行微调，使模型学习子技能之间的时序衔接。

推荐写法：

[
\mathcal{L}_{final}
===================

\mathcal{L}*{full}
+
\lambda*{replay}\mathcal{L}*{sub}
+
\lambda_s\mathcal{L}*{smooth}
]

其中：

* (\mathcal{L}_{full})：完整饮品服务任务轨迹损失；
* (\mathcal{L}_{sub})：子技能 replay/rehearsal 损失；
* (\lambda_{replay})：子技能回放权重；
* (\mathcal{L}_{smooth})：动作平滑项。

请强调：完整轨迹微调时保留部分子任务数据进行 replay，可以降低后续训练覆盖前期子技能的问题。

这部分可以概括为：

**先让模型学会每个动作，再让模型学会什么时候做哪个动作，最后让模型学会把这些动作连成完整任务流。**

其中：

* (\theta_{base})：GR00T 原始基础能力参数；

请注意表达边界：

## 9. 原理支撑

请从以下角度论证方法合理性：

1. **课程学习思想**
   子技能到完整任务的训练流程符合由易到难的课程学习思想。

2. **持续学习与防遗忘思想**
   如果按顺序训练多个子技能，后训练技能可能覆盖前面技能。采用混合采样和 replay/rehearsal 可以降低灾难性遗忘。

3. **条件化多任务学习思想**
   task role token 使同一模型能够根据不同任务角色输出不同动作模式。

4. **动作子空间加权监督思想**
   双臂任务中不同阶段的主执行臂不同，因此不应对所有动作维度平均监督，而应根据任务角色动态调整监督权重。

5. **工程可诊断性**
   分项 loss 能够帮助判断失败来源，例如右臂抓取失败、右臂倒水失败、左臂抓杯失败或双臂协同失败。

## 12. 最终要强调的创新点

请将创新点凝练为三点：

1. **基于任务流程先验的弱监督阶段标注**
   将长时序双臂饮品服务任务划分为可学习、可诊断的子技能片段。

3. **子技能均衡采样与 BTA 加权损失**
   通过混合采样、角色感知动作权重和完整轨迹 replay 微调，提高长时序双臂任务训练的稳定性与可诊断性。

## 14. 一句话总结

本文方法的核心不是重新提出一个 VLA 大模型，而是在 GR00T 类基础模型之上，引入面向双臂饮品服务任务的轻量级任务感知适配机制。通过弱监督阶段标注、task role token、子技能均衡训练和 BTA 加权损失，使模型从单一整体 action chunk 模仿学习，转向角色明确、阶段清晰、动作子空间可诊断的结构化双臂模仿学习。
